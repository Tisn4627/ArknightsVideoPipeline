## 1. 安装与启动

### 1.1 环境要求

- **操作系统**：Windows 10 / 11（本开发环境）
- **Python**：3.12 或更高版本
- **ffmpeg / ffprobe**：需要在系统 PATH 中可用
- **MAA（MaaAssistantArknights）**：项目目录内需包含可识别的 MAA 目录（如 `MAA-v5.12.1-win-x64`）

### 1.2 安装依赖

推荐使用项目自带的虚拟环境：

```powershell
# 进入项目目录
cd ArknightsVideoPipeline-dev

# 创建虚拟环境（如尚未创建）
python -m venv venv

# 激活虚拟环境
.\venv\Scripts\activate

# 安装依赖（包含 PyQt6 GUI 库）
pip install -r requirements.txt
```

> 安装 PyQt6 时可能需要下载约 80MB 的 Qt 运行库，请保持网络畅通。

### 1.3 启动 GUI

```powershell
# 在虚拟环境中执行
python gui.py
```

启动成功后将弹出主窗口。如果习惯命令行，CLI 入口 `python main.py` 仍然保留且功能不变。

## 2. 主界面组成

启动后主窗口参考 Material Design 3 官网风格布局，分为几个区域：

| 区域 | 位置 | 内容 |
|------|------|------|
| Navigation Rail（导航栏） | 窗口最左侧 | Home / Settings / Help 入口，图标+标签；窗口较窄时自动折叠为仅图标 |
| Hero 区域 | Home 页内容区顶部 | 应用大标题 "Arknights Video Pipeline"、描述文字、"Get started" 主按钮 |
| 配置卡片 | Home 页卡片网格左侧 | 输入视频、背景板图片、MAA 路径、输出目录、合成风格、日志级别、跳过步骤、操作按钮 |
| 步骤卡片 | Home 页卡片网格右侧 | 5 个流水线步骤状态展示 |
| 进度卡片 | Home 页卡片网格下方 | 当前处理进度与状态文字 |
| 日志卡片 | 进度卡片下方 | 运行日志，支持自动滚动、复制、清空 |
| 底部信息区 | Home 页内容区最下方 | 使用提示与功能说明 |
| 设置页面 | 点击导航栏 Settings 后显示 | 外观主题切换（实时预览）+ 配置文件按类型生成 |

窗口默认大小为 1360×900，最小大小为 800×640。当窗口宽度小于 960px 时，左侧导航栏会自动折叠为仅图标模式；内容区始终保持可滚动，确保小窗口下也能正常操作。点击导航栏的 **Settings** 会切换到独立的设置页面，**Home** 返回主页，**Help** 弹出关于对话框（不离开当前页面）。

## 3. 功能模块操作说明

### 3.1 选择输入视频

1. 在左侧配置区找到 **输入视频**；
2. 点击右侧的 **浏览** 按钮；
3. 在弹出的文件对话框中选择游戏录像文件；
4. 支持的格式：`.mp4`、`.avi`、`.mkv`、`.mov`、`.flv`、`.wmv`。

> 选择视频后，程序会自动验证文件是否存在、格式是否支持，以及能否被 ffprobe 正常解析。

### 3.2 选择背景板图片

1. 在左侧配置区找到 **背景板图片**；
2. 点击 **浏览** 选择背景板图片；
3. 支持的格式：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.webp`。

> **注意**：只有选择 **style1** 合成风格时才必须提供背景板图片。切换到 **style2** 后，背景板图片输入框会自动禁用并清空。

### 3.3 设置 MAA 路径

1. 在左侧配置区找到 **MAA 路径**；
2. 点击 **浏览** 选择 MAA 项目目录（包含 `MAA.exe` 或 `MaaCore.dll` 的文件夹）。

> 如果留空，程序将使用 `config/pipeline.json` 中配置的 `maa_path`。若路径无效，运行时会提示错误。

### 3.4 设置输出目录

1. 在左侧配置区找到 **输出目录**；
2. 点击 **浏览** 选择用于存放处理结果的文件夹。

> 若留空，默认输出到 `output/<视频名>/` 目录下。

### 3.5 选择合成风格

在 **合成风格** 下拉框中选择：

| 风格 | 说明 | 是否需要背景板 |
|------|------|----------------|
| **style1** | 底板图片 + 视频叠加，文本显示于视频区域旁 | 是 |
| **style2** | 全屏视频 + 底部居中字幕 | 否 |

### 3.6 设置日志级别

在 **日志级别** 下拉框中选择：

- **DEBUG**：最详细，包含每帧匹配结果
- **INFO**：常规运行信息（推荐）
- **WARNING**：仅警告和错误
- **ERROR**：仅错误

日志会同时显示在界面下方的 **运行日志** 区域，并写入 `output/<视频名>/pipeline.log` 文件。

### 3.7 跳过步骤

如果只想执行部分步骤，可以勾选对应的复选框。复选框以 3 列网格排列，共两行：

| 复选框 | 对应步骤 |
|--------|----------|
| MAA | 视频转 MAA 作业 JSON |
| Formation | 编队配置转文本 |
| Actions | 操作指令转文本 |
| Track | 识别开始按钮时间戳 |
| Compose | 视频合成 |

> 窗口宽度较小时，该网格会自动适应，避免复选框被截断。

### 3.8 验证输入

点击 **验证输入** 按钮，程序会检查：

- 视频文件是否存在、格式是否支持、是否能被 ffprobe 解析；
- style1 风格下背景板图片是否存在；
- MAA 路径是否存在（如果已填写）。

如果存在问题，会弹出提示框列出所有错误；通过则提示 "输入校验通过"。

### 3.9 开始处理

配置完成后，点击 **开始处理**：

1. 右侧 **处理进度** 会显示当前步骤与总进度；
2. **运行日志** 会实时输出处理信息；
3. 5 个步骤卡片会依次显示 "运行中"、"已完成" 或 "失败"。

处理过程中，界面不会卡顿，您可以继续查看日志或取消任务。

### 3.10 取消处理

处理过程中，点击 **取消** 按钮即可请求停止。由于底层 MAA 识别、OpenCV 匹配等操作无法安全强制中断，取消请求会在当前步骤结束后生效。

### 3.11 使用左侧导航栏

窗口最左侧是精简后的 Material Design 3 风格 Navigation Rail：

| 图标 | 标签 | 功能 |
|------|------|------|
| 🏠 | Home | 返回主页面（默认选中） |
| ⚙ | Settings | 切换到设置页面（主题 + 配置文件生成） |
| ? | Help | 点击显示关于对话框（不离开当前页面） |

当窗口宽度小于 960px 时，导航栏会自动折叠，只显示图标；将鼠标悬停或重新拉大窗口即可恢复图标+标签。

### 3.12 设置页面

点击导航栏的 **Settings** 进入设置页面。页面包含两张卡片：

#### 3.12.1 外观（Appearance）—— 主题切换

- 卡片右侧有一个 Material Design 3 风格的开关 **Dark theme**；
- 拨动开关即可在浅色与深色主题之间切换，**效果即时生效**（实时预览）：Navigation Rail、Hero 副标题、卡片、按钮等所有控件颜色会立即同步更新；
- 再次进入设置页时，开关会自动反映当前主题状态。

#### 3.12.2 配置文件（Configuration files）—— 按类型生成

- 卡片列出 6 种配置文件类型，每个复选框标注名称与目标文件：

  | 选项 | 目标文件 | 说明 |
  |------|----------|------|
  | Pipeline | `pipeline.json` | 主流水线配置 |
  | Formation | `formation.json` | 编队配置 |
  | Actions | `actions.json` | 操作指令配置 |
  | Track | `track.json` | 开始按钮识别配置 |
  | Compose · style1 | `video_compose/style1.json` | style1 视频合成配置 |
  | Compose · style2 | `video_compose/style2.json` | style2 视频合成配置 |

- 勾选需要的类型（可点击 **Select all** 全选、**Clear** 清空）；
- 点击 **Generate**，程序会按所选类型在 `config/` 目录下生成（或重置为）默认配置文件，与 CLI 中执行 `python main.py --init-config <module>` 行为一致；
- 生成结果会显示在按钮右侧的状态文本中：成功显示生成数量，失败会用红色标出失败项；未选择任何类型时提示错误。

> 提示：首次使用前建议在设置页生成全部默认配置，确保 `config/` 目录下配置文件完整。

点击设置页底部的 **Back to home**（或导航栏的 **Home**）即可返回主页。

## 4. 处理流程示例

### 示例 1：style1 完整处理

1. 点击 **输入视频** 旁的浏览，选择 `video.mp4`；
2. 点击 **背景板图片** 旁的浏览，选择 `bg.png`；
3. （可选）设置 **MAA 路径** 与 **输出目录**；
4. 确认 **合成风格** 为 `style1`；
5. 点击 **验证输入**，确认无错误；
6. 点击 **开始处理**，等待处理完成；
7. 在输出目录中查看生成的 `output_video.mp4` 与报告文件。

### 示例 2：style2 无需背景板

1. 选择输入视频；
2. 将 **合成风格** 切换为 `style2`；
3. 此时 **背景板图片** 会自动禁用；
4. 点击 **开始处理**。

### 示例 3：仅运行 MAA 识别

1. 选择输入视频；
2. 勾选 **跳过步骤** 中的 "编队"、"操作"、"按钮"、"合成"；
3. 点击 **开始处理**；
4. 只生成 Copilot JSON 文件后结束。

## 5. 常见问题与解决办法

### Q1：启动 GUI 时提示 "No module named 'PyQt6'"

**原因**：虚拟环境中未安装 PyQt6。

**解决**：激活虚拟环境后执行：

```powershell
pip install PyQt6>=6.6.0
```

或直接使用：

```powershell
pip install -r requirements.txt
```

---

### Q2：点击 "开始处理" 后提示 "视频文件不存在"

**原因**：输入视频路径为空或文件已被移动/删除。

**解决**：重新点击 **浏览** 选择有效的视频文件，并确保文件路径中没有特殊字符或中文乱码。

---

### Q3：style1 提示 "需要背景板图片"

**原因**：style1 合成风格必须在视频画面上叠加背景板图片。

**解决**：
- 选择一张背景板图片；或
- 将 **合成风格** 切换为 `style2`（全屏视频 + 字幕，无需背景板）。

---

### Q4：MAA 识别步骤失败

**可能原因**：
- MAA 路径未配置或配置错误；
- 输入视频分辨率、帧率不符合 MAA 要求；
- MAA 依赖的运行库缺失。

**解决**：
1. 确认 MAA 路径指向包含 `MAA.exe` / `MaaCore.dll` 的目录；
2. 查看 **运行日志** 中的具体错误信息；
3. 尝试使用 CLI 运行相同命令，对比错误输出：`python main.py video.mp4 -b bg.png`。

---

### Q5：日志区域没有输出

**原因**：日志级别设置过高，或日志尚未产生。

**解决**：将 **日志级别** 调整为 **INFO** 或 **DEBUG**，然后重新开始处理。

---

### Q6：取消按钮没有立即生效

**原因**：当前正在执行的 MAA 识别、OpenCV 模板匹配或视频合成等操作无法被安全强制中断。

**解决**：取消请求会在当前步骤边界生效。如果任务长时间无响应，可以关闭窗口强制退出，但这可能导致临时文件残留。

---

### Q7：界面显示字体异常或出现 "Cannot find font directory" 提示

**原因**：在无头服务器或某些精简 Windows 环境中，Qt 6 未能找到字体目录。

**解决**：
- 在普通 Windows 桌面环境中运行时通常不会出现此问题；
- 若仍出现，可安装常用字体（如 Microsoft YaHei、Consolas），或在项目根目录创建 `fonts/` 目录并放入 TTF 字体文件。

---

### Q8：配置没有保存

**原因**：程序仅在正常关闭窗口时保存当前配置；强制结束进程可能导致配置丢失。

**解决**：点击窗口右上角关闭按钮正常退出。配置会保存到 `config/pipeline.json`。

## 6. 快捷键与提示

| 操作 | 说明 |
|------|------|
| 导航栏 → Home | 返回主页面 |
| 导航栏 → Settings | 进入设置页面（主题切换 / 配置文件生成） |
| 导航栏 → Help | 查看关于对话框 |
| 右键日志区域 | 复制 / 清空日志 |
| 拖动分隔条 | 调整左右或上下区域大小 |

**使用建议**：

- 首次使用前，建议进入 **Settings → Configuration files**，点击 **Select all** 后 **Generate**，确保 `config/` 目录下配置文件完整；
- 处理前务必点击 **验证输入**，提前发现路径或格式问题；
- 长时间运行任务时，保持电脑插电并避免休眠。
